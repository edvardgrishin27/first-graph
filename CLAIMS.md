# CLAIMS.md — каждое число можно перезапустить

В ролике звучат числа. Здесь для каждого из них лежит команда, которой это число получено, и настоящий вывод этой команды.

Смысл файла простой. Волна graph engineering раздала много цифр и ни одной команды, которой их можно перепроверить. Мы за это критикуем — значит, у себя обязаны сделать наоборот. Если вы читаете это и думаете «а точно ли», не верьте: скопируйте команду и запустите.

**Как читать.** Все команды выполняются из корня репозитория. Столбец «Что получилось» — настоящий вывод прогона, сокращённый до значимых строк. Ничего не переписано из README: README тоже проверялся этими командами, и одно расхождение в нём нашлось (раздел 8).

**Условия замера.** 22 июля 2026, коммит `78e5bf9` плюс добавленный этой проверкой `examples/apk-release.json`. macOS 26.5, Python 3.14.5, внешних зависимостей нет — только `pytest` для тестов. На другой машине числа времени не поменяются: они считаются из описания пайплайна, а не замеряются секундомером.

**Статусы:**

| Статус | Что означает |
|---|---|
| **подтверждено** | команда выполнена, вывод совпал с утверждением |
| **расходится** | команда выполнена, вывод другой; подробности в разделе 8 |
| **внешний источник** | репозиторий этого не доказывает и не может; доказательство лежит снаружи |
| **не проверяется репозиторием** | утверждение о том, чего в репозитории пока нет |

| Тот, кто выстроит визуальный план после сценария, **зря прождёт 40 единиц** | скрипт А (ниже) + скрипт Б (ниже) | ветка действительно ждёт ровно 40 (старт 32 → старт 72), **но итог пайплайна не меняется: 129 в обоих случаях** | **расходится** — см. 8.2 |
| Ожидание humanizer'а за фактчеком стоит реальных 15 единиц | скрипт А | `по графу: 129.0` → `humanizer ждёт фактчек: 144.0` | подтверждено |

**Скрипт А — цена двух наивных перестроений.** Берём тот же файл и подвешиваем один шаг за чужой выход, как сделал бы человек, набирающий пайплайн строкой:

```python
import json

from first_graph.arrows import Report, audit, from_dicts

ФАЙЛ = "examples/news-release.json"
шаги: list[dict] = json.load(open(ФАЙЛ, encoding="utf-8"))["steps"]


def подвесить(кого: str, на_что: list[str]) -> Report:
    """Тот же пайплайн, но один шаг ждёт другой вход — как сделал бы человек."""
    другие = [dict(ш, reads=на_что) if ш["id"] == кого else ш for ш in шаги]
    return audit(from_dicts(другие))


print("по графу:                      ", audit(from_dicts(шаги)).parallel_time)
print("визуальный план ждёт сценарий: ", подвесить("визуальный_план", ["черновик"]).parallel_time)
print("humanizer ждёт фактчек:        ", подвесить("humanizer", ["правки"]).parallel_time)
```

```
по графу:                       129.0
визуальный план ждёт сценарий:  129.0
humanizer ждёт фактчек:         144.0
```

**Скрипт Б — во сколько ветка может начаться.** Здесь мы лезем во внутреннюю функцию `_real_dependencies`: это диагностика для проверки утверждения, а не публичный API.

```python
import json

from first_graph.arrows import _real_dependencies, from_dicts

шаги: list[dict] = json.load(open("examples/news-release.json", encoding="utf-8"))["steps"]


def старт(сырые: list[dict], кого: str) -> float:
    """Во сколько шаг реально может начаться."""
    разобранные = from_dicts(сырые)
    зависимости = _real_dependencies(разобранные)
    по_id = {ш.id: ш for ш in разобранные}
    конец: dict[str, float] = {}
    for ш in разобранные:
        начало = max((конец[р] for р in зависимости[ш.id]), default=0.0)
        конец[ш.id] = начало + по_id[ш.id].duration
        if ш.id == кого:
            return начало
    raise KeyError(кого)


наивно = [dict(ш, reads=["черновик"]) if ш["id"] == "визуальный_план" else ш for ш in шаги]
print("старт визуального плана по графу:", старт(шаги, "визуальный_план"))
print("старт, если ждёт сценарий:       ", старт(наивно, "визуальный_план"))
```

```
старт визуального плана по графу: 32.0
старт, если ждёт сценарий:        72.0
```

## 3. Отклонённый кандидат: сборка мобильного приложения

Сборка APK предлагалась главным героем ролика. Её померили тем же инструментом и отказались — это самое слабое число из трёх кандидатов, и ставить его на самое видное место было бы враньём себе.

| Утверждение | Команда | Что получилось | Статус |
|---|---|---|---|
| Релиз приложения даёт всего **1.2×** — 45 против 38 | `python3 -m first_graph arrows examples/apk-release.json` | `Было 45` / `Стало 38` / `Ускорение 1.2×` | подтверждено на модели, см. оговорку |
| В нём **3 фальшивых стрелки из 6** и **5 уровней** | там же | `ИТОГ: фальшивых стрелок 3 из 6`; уровней ровно 5 | подтверждено на модели |
| Причина слабого числа: хвост `сборка → подпись → публикация` распараллелить нельзя физически | там же | уровни 3, 4, 5 — по одному шагу; критический путь упирается в 25 минут gradle | подтверждено |

```
ИТОГ: фальшивых стрелок 3 из 6

КАК ЭТО ДОЛЖНО ИДТИ НА САМОМ ДЕЛЕ
  уровень 1: зависимости, changelog  ← одновременно
  уровень 2: типы, тесты  ← одновременно
  уровень 3: сборка_apk
  уровень 4: подпись
  уровень 5: публикация

Было (по очереди):     45
Стало (по графу):      38
Ускорение:             1.2×
```

**Оговорка, без которой это число нечестное.** Файла с исходной моделью сборки APK в репозитории не было — цифра 1.2× существовала только в плане. Модель восстановлена по её описанию (гейт на тестах, последовательный хвост) и положена в `examples/apk-release.json`, чтобы утверждение вообще можно было перезапустить. Совпадение с планом (1.2×, 3 из 6, 5 уровней) — приятное, но конкретная кратность зависит от выбранных длительностей: поставьте gradle 40 минут вместо 25, и она уедет. Устойчиво здесь другое, и говорить в ролике надо именно это: **у релизного конвейера три последних уровня — по одному шагу, и никакой граф этого не изменит.**

**Скрипт Г:**

```python
from first_graph import GraphError, State, fan_out, gate, router


def падает(состояние: State) -> State:
    raise RuntimeError("ветка упала")


def работает(состояние: State) -> State:
    return {**состояние, "ok": True}


print("веер:  ", fan_out({}, [работает, падает, работает]))

try:
    router({}, lambda s: "нет-такого", {"а": работает})
except GraphError as ошибка:
    print("роутер:", ошибка)

попытки: list[int] = []


def строит(состояние: State) -> State:
    попытки.append(1)
    return состояние


_, прошло = gate({}, build=строит, verify=lambda s: False, max_attempts=3)
print("ворота: попыток", len(попытки), "| прошло", прошло)
```

```

роутер: роутер вернул «нет-такого», но такого маршрута нет и запасной не задан (есть: а)
ворота: попыток 3 | прошло False
```

**Скрипт Д — независимая проверка «только stdlib»** (не доверяет собственному тесту и сверяется со списком модулей самого Python):

```python
import ast
import pathlib
import sys

имена: set[str] = set()
for файл in sorted(pathlib.Path("first_graph").glob("*.py")):
    for узел in ast.walk(ast.parse(файл.read_text(encoding="utf-8"))):
        if isinstance(узел, ast.Import):
            имена |= {a.name.split(".")[0] for a in узел.names}
        elif isinstance(узел, ast.ImportFrom):
            имена.add((узел.module or "").split(".")[0])

свои = {"first_graph"}
внешние = sorted(и for и in имена - свои if и and и not in sys.stdlib_module_names)
print("все импорты ядра:", ", ".join(sorted(имена - свои)))
print("вне stdlib:", внешние or "нет")
```

```
вне stdlib: нет
```

## 8. Расхождения, найденные этой проверкой

### 8.1. README показывает демо не тем выводом

В README в блоке «Попробовать за тридцать секунд» под командой `python3 -m first_graph demo` напечатано `ИТОГ: фальшивых стрелок 6 из 8`. Настоящий вывод — `4 из 6`.

Причина: в `DEMO_PIPELINE` (`first_graph/cli.py`) семь шагов и пять проверок, а «6 из 8» — это числа ночного аудита из `examples/night-audit.json`, где проверок семь. Числа времени в том же блоке (32 / 14 / 2.3×) верные и относятся именно к демо.

Что с этим делать: поправить строку в README до `4 из 6`. Числа `6 из 8` остаются законными — но рядом с `night-audit.json`, а не с `demo`.

### 8.2. «Зря прождут 40 единиц» — правда про ветку, неправда про итог

Ветка визуального плана при наивной раскладке действительно простаивает ровно 40 единиц: старт сдвигается с 32 на 72 (скрипт Б). Но общее время пайплайна при этом **не меняется — 129 в обоих случаях** (скрипт А): у этой ветки есть запас, критический путь идёт через `сценарий → фактчек → финал → описание → сборка`.

Как говорить честно: «визуальный план сорок единиц стоит и ждёт того, чего ему не нужно» — верно. «Это стоит вам сорока единиц выпуска» — неверно.

Соседняя находка, наоборот, стоит настоящих денег: если подвесить humanizer за фактчек, пайплайн растёт со 129 до **144**, то есть на 15 единиц. Если нужно одно число про цену лишнего ожидания — брать надо это.

### 8.3. Формулировку про Graphify надо сузить

«У Graphify нет JSON-вывода» — слишком широко и на факт-чеке развалится: `--json` в CLI есть, у `diagnose multigraph` и `god-nodes`. Проверяемая формулировка: «у `path`, `explain` и `query` вывода в JSON нет — только текст для человека» (проверено на локально установленном `graphifyy` 0.9.23; в другой версии может отличаться, версию надо называть вслух).

## Как проверить всё разом

Одна команда из корня репозитория. Тесты, все четыре пайплайна и якорь:

```bash
python3 -m pytest tests/ -q | tail -1; python3 -m first_graph demo | tail -3; for f in examples/*.json; do echo "--- $f"; python3 -m first_graph arrows "$f" | tail -3; done; python3 examples/anchor_demo.py | tail -2
```

Настоящий вывод:

```
64 passed in 2.81s
Было (по очереди):     32
Стало (по графу):      14
Ускорение:             2.3×
--- examples/apk-release.json
Было (по очереди):     45
Стало (по графу):      38
Ускорение:             1.2×
--- examples/news-release.json
Было (по очереди):     250
Стало (по графу):      129
Ускорение:             1.9×
--- examples/night-audit.json
Было (по очереди):     39
Стало (по графу):      14
Ускорение:             2.8×

  Агенты согласились. Реальность — нет.
```

Ненулевые коды возврата здесь штатные: `arrows` возвращает `1`, когда нашёл что распараллелить, а `anchor_demo.py` — `1`, потому что граф и должен встать. Именно так это и проверяется в CI (`.github/workflows/ci.yml`).
