# Эхо-камера

**Соглашается ли модель-ревьюер со своим же кодом.**

Весь корпус статей про graph engineering повторяет одно и то же: «модели
предпочитают собственные ответы, поэтому агент-ревьюер соглашается с
агентом-исполнителем — ставьте разные модели». Тезис правдоподобный. Ни в
одной из этих статей он не измерен.

Здесь он измеряется. Или не измеряется — если эффекта на этих данных нет,
скрипт так и напечатает.

Ядро репозитория при этом не тронуто: `first_graph/*.py` остаётся в своих
800 строках без внешних зависимостей, весь эксперимент живёт в этой папке.

---

## Что меряем

Доля случаев, когда судья **одобрил код, в котором тесты нашли изъян**.
Чем выше доля — тем чаще судья не заметил ошибку.

План 2×2:

|                    | судья видит **весь контекст** исполнителя | судья получает **чистый контекст** (только код) |
|--------------------|-------------------------------------------|--------------------------------------------------|
| судья на **той же** модели, что писала код | **A** | **B** |
| судья на **другой** модели                 | **C** | **D** |

**Главное сравнение — B против D.** Обе ячейки с чистым контекстом: судья
видит спецификацию и код, без истории, без отчёта исполнителя, без указания
авторства. Различие ровно одно — та это модель или другая. Всё остальное —
побочные сравнения, и в отчёте они помечены как побочные.

Сравнение A против C соблазнительнее, но грязнее: там вместе с моделью
меняется и объём информации в контексте.

## Почему именно так

**Код с изъяном производит сама модель.** Подложить чужой баг было бы проще,
но самопредпочтение — про свою работу. Оценка чужого дефекта не говорит о нём
ничего. Поэтому исполнителю даётся задача, где легко ошибиться, и мы работаем
с тем, что он написал.

**Правду устанавливает `anchor()`, а не ещё одна модель.** Код и тест
кладутся во временную папку, запускается настоящий pytest, решает код
возврата. Если бы наличие изъяна определяла третья модель, мы измеряли бы
согласие моделей между собой — то есть ровно то, в чём и подозреваем
систематическую ошибку.

**План парный.** Один и тот же артефакт судят все четыре ячейки. Поэтому
разницу между B и D нельзя списать на то, что одной ячейке достались задачи
полегче: им достался буквально один и тот же код.

**Чистый код тоже судится — это контроль на снисходительность.** Судья,
который принимает вообще всё, покажет высокую долю одобрений на изъянном коде
безо всякого самопредпочтения. Без контрольной таблицы этого не увидеть.
Отключается флагом `--only-flawed`, если жалко денег.

**Судья видит спецификацию, но никогда не видит тест.** Без спецификации
«изъян» неотличим от «мне не сказали, что так нельзя». С тестом на руках
задача становится тривиальной.

**Системный промпт судьи один и тот же во всех четырёх ячейках.** Иначе
менялись бы два фактора сразу.

**Температура: у исполнителя 1.0, у судьи 0.0.** Исполнителю нужен разброс
решений, судье — повторяемость.

## Как запустить

Сначала — сухой прогон. Он проходит весь конвейер на заглушках вместо вызовов
API: ни одного обращения к модели, ни копейки расхода.

```bash
python3 -m experiments.echo --dry-run
```

Что в сухом прогоне настоящее: извлечение кода из ответа, запуск pytest
якорем, разбор вердиктов, сведение таблиц, статистика. Заглушены только
вызовы моделей.

Что в сухом прогоне ненастоящее и намеренно: **судья-заглушка бросает честную
монету, одинаково во всех четырёх ячейках**. Она не подыгрывает тезису. Если
бы подыгрывала, `--dry-run` печатал бы красивую таблицу с «подтверждённым»
эффектом, и однажды её приняли бы за результат. Поэтому на сухом прогоне
нормальный вывод — «эффект не воспроизвёлся».

Посмотреть задачи и их ловушки:

```bash
python3 -m experiments.echo --list-tasks
```

Настоящий прогон (нужны `ANTHROPIC_API_KEY` и/или `OPENAI_API_KEY`
в окружении; идентификаторы моделей сверьте с провайдером — они меняются
чаще этого файла):

```bash
python3 -m experiments.echo --runs 8 \
    --executor anthropic:МОДЕЛЬ-1 --other openai:МОДЕЛЬ-2
```

И **обязательно зеркальный прогон** — с переставленными моделями:

```bash
python3 -m experiments.echo --runs 8 \
    --executor openai:МОДЕЛЬ-2 --other anthropic:МОДЕЛЬ-1
```

Почему обязательно — в угрозах валидности, пункт 1. Коротко: в одном прогоне
«своя модель» неотличима от «эта модель просто придирчивее».

Сырые данные каждого настоящего прогона (все промпты, все ответы, все
вердикты) складываются в `experiments/echo/runs/*.json`. Если число из
прогона куда-то попадёт — этот файл и есть доказательство, его стоит
сохранить рядом.

## Задачи

Четыре, все маленькие: токены стоят денег.

| id | функция | ловушка |
|---|---|---|
| `скидка` | `цена_со_скидкой` | встроенный `round()` округляет половину к чётному, а не вверх |
| `раздать` | `раздать` | деление с округлением теряет копейки: сумма частей не сходится |
| `интервалы` | `слить_интервалы` | привычное `<=` сливает касающиеся полуоткрытые интервалы |
| `обрезать` | `обрезать` | срез строки считает символы, а предел задан в байтах |

Каждая ловушка проверена тестом репозитория в обе стороны: верное решение
проходит тест задачи, решение с ловушкой — падает
(`tests/test_echo_experiment.py`). Если бы тест не ловил ловушку, эксперимент
мерил бы не то.

## Что печатает

Кусок настоящего вывода `--dry-run` (цифры получены на честной монете и
поэтому ничего не значат — они здесь, чтобы показать форму отчёта):

```
ЧТО ПРОИЗВЕЛА МОДЕЛЬ (правду установил pytest, а не мнение)
  скидка         с изъяном 3 из 5
  раздать        с изъяном 3 из 5
  интервалы      с изъяном 5 из 5
  обрезать       с изъяном 3 из 5
  ИТОГО          с изъяном 14 из 20

ДОЛЯ ОДОБРЕНИЙ КОДА С ИЗЪЯНОМ — чем выше, тем чаще судья не заметил изъян
                      общий контекст            чистый контекст
  та же модель        A  7/14 =  50%            B  5/14 =  36%
  другая модель       C  4/14 =  29%            D  7/14 =  50%

ГЛАВНОЕ СРАВНЕНИЕ: B против D
  обе с чистым контекстом, различие только в модели судьи
  B (та же модель + чистый контекст): 5/14 = 36% [95% 16–61]
  D (другая модель + чистый контекст): 7/14 = 50% [95% 27–73]
  разница: -14.3 п.п.
  пар: 14; расхождений: B одобрила, D — нет — 1; наоборот — 3
  точный тест Макнемара: p = 0.625

ВЫВОД
  Эффект не воспроизвёлся: разница -14.3 п.п. неотличима от случайной (p =
  0.625 при пороге 0.05).
  Тезис «ревьюер соглашается со своим же кодом» на этих данных не
  подтверждён.
```

Статистика — точный тест Макнемара по парам (план парный, значит работают
только расхождения внутри пар) и интервалы Уилсона для долей. Всё на
стандартной библиотеке: `math.comb` хватает, ни numpy, ни scipy не нужны.

Ветка «эффект не воспроизвёлся» написана так же подробно, как ветка
«воспроизвёлся», и есть отдельная ветка для эффекта в обратную сторону.
Порог и главное сравнение заданы заранее, в коде, а не выбираются по факту.

## Угрозы валидности

Их несколько, и честнее перечислить все.

1. **Один прогон не различает «своя модель» и «эта модель строже».** В B и D
   сидят разные модели, поэтому разница может быть просто их разной
   придирчивостью. Лечится зеркальным прогоном (`--executor` и `--other`
   местами) и сравнением обоих. Без второго прогона результат не
   интерпретируется как самопредпочтение. Это главная угроза.
2. **Чистый контекст — не анонимность.** Мы убрали историю и авторство, но не
   стилевой отпечаток: модель может узнать собственную манеру писать код.
   Тогда «чистый контекст» окажется слабее, чем задумано.
3. **Задачи маленькие и специфические.** Четыре ловушки на границах — это не
   ревью большого модуля с архитектурными решениями. Переносимость вывода на
   настоящее ревью в графе не установлена.
4. **Артефакты внутри задачи не независимы.** Одна и та же ловушка
   повторяется `--runs` раз; эффективный размер выборки меньше числа
   артефактов. Тест Макнемара этой кластеризации не учитывает — значит, p
   оптимистичнее правды.
5. **«Изъян» неоднороден.** Тест валит и грубую ошибку в деньгах, и формальное
   расхождение (список вместо кортежа), и синтаксическую поломку. Пропустить
   формальное расхождение — не то же самое, что не заметить потерянную
   копейку, а в таблице это одна строка.
6. **Спецификация могла быть неполной.** Тогда изъян по тесту есть, а судья
   не мог о нём знать, и мы засчитываем ему «не заметил» несправедливо.
   Смягчено тем, что спецификация одна и та же у исполнителя и судьи и
   описывает граничные случаи явно; проверяется глазами по сохранённому JSON.
7. **Общий контекст меняет сразу два фактора** — авторство и объём информации:
   в истории есть пояснения исполнителя, которых нет в чистом контексте.
   Поэтому A и C нельзя читать как чистый фактор «своё/чужое».
8. **Температура судьи 0 — одно суждение на артефакт.** Разброс между
   повторными суждениями об одном и том же коде не измеряется вовсе. Прогон
   с температурой судьи выше нуля — другой эксперимент, не этот.
9. **Множественные сравнения.** Главное объявлено заранее (B против D),
   побочные печатаются без поправки на множественность. Читать их как
   доказательства нельзя.
10. **Отказ от вердикта исключается из подсчёта.** Если такие отказы
    распределены по ячейкам неслучайно (например, своя модель чаще уходит от
    прямого ответа) — это смещение, которого таблица не видит. Число отказов
    печатается в разделе «СБОИ».
11. **Судья мог заметить изъян и всё равно принять**, посчитав его
    несущественным. Бинарный вердикт этого не различает.
12. **Идентификатор модели — не фиксированные веса.** Провайдер обновляет
    модель под тем же именем; повтор через полгода может дать другое число не
    из-за нас.
13. **Слой вызовов API живьём не проверялся** — там, где он писался, не было
    ни ключей, ни сети. Проверен только путь через заглушку. Форматы запросов
    взяты из документации и могли устареть; всё, что может разойтись,
    собрано в константах наверху `providers.py`.
14. **Мы запускаем код, который написала модель, без песочницы.** Временная
    папка и таймаут — не изоляция. Это риск исполнения, а не валидности, но
    молчать о нём нельзя.

## Сколько стоит

Число вызовов считается точно:

```
вызовов = задачи × повторов × (1 исполнитель + 4 судьи)
```

На значениях по умолчанию (4 задачи × 5 повторов): 20 вызовов исполнителя и
80 судейских, итого **100 вызовов** на прогон. С `--only-flawed` судейских
станет меньше — ровно во столько раз, какая доля кода оказалась изъянной.

Размеры промптов замерены (в знаках, а не в токенах — токенизатор у каждого
свой):

| что | знаков на вызов |
|---|---|
| задание исполнителю | 640–790 |
| судья, чистый контекст | 1180–1650 |
| судья, общий контекст | плюс длина ответа исполнителя целиком |

Дальше — **оценка, а не замер**: настоящих вызовов API не было. Для
кириллицы с кодом счёт примерно втрое меньше знаков, ответы судьи короткие.
Порядок для прогона по умолчанию — около 50–60 тысяч входных и 20–30 тысяч
выходных токенов, разложенных между двумя моделями. На средних моделях это
десятки центов, на флагманах — единицы долларов. Зеркальный прогон удваивает
и то и другое, `--runs` масштабирует линейно.

Настоящие числа печатает сам скрипт, в разделе «РАСХОД», с разбивкой по узлам
графа. В деньги их переводит дополнение репозитория — со **своим** прайсом и
обязательной датой проверки тарифов:

```bash
python3 -m experiments.echo --runs 8 --prices мой-прайс.json
```

Формат прайса — `first_graph/extras/prices.example.json`. Цены в этот
эксперимент не зашиты намеренно: счёт без даты тарифов — невоспроизводимое
число, ровно то, за которое принято критиковать чужие README.

## Что в папке

| файл | зачем |
|---|---|
| `tasks.py` | четыре задачи: спецификация, тест-ловушка, заглушки для сухого прогона |
| `prompts.py` | промпты обеих ролей и разбор ответов (вердикт, извлечение кода) |
| `providers.py` | тонкий слой над HTTP-API: Protocol, Anthropic, OpenAI, только urllib |
| `stubs.py` | заглушка вместо API; судья с честной монетой |
| `truth.py` | правда об изъяне через `anchor()` + предполётная проверка pytest |
| `runner.py` | конвейер: произвести, проверить, собрать вердикты по четырём ячейкам |
| `stats.py` | Макнемар и Уилсон на `math.comb` |
| `report.py` | таблицы и вывод, включая вывод «не получилось» |
| `__main__.py` | командная строка |

Тесты каркаса — в `tests/test_echo_experiment.py`, запускаются вместе со
всеми остальными:

```bash
python3 -m pytest tests/ -q
```
